Methodology — NMCP Malaria EPI Reporting Status Tool

1 · Facility Registry

All Level-5 organisation units are retrieved from the DHIS2 instance (sl.dhis2.org/hmis23) using the /api/organisationUnits endpoint with ancestor fields to extract district (Level-2) and chiefdom (Level-4) hierarchy. Each facility is classified by the last word of its display name using a suffix-first rule (getHFType()). Only the five assessed types are retained:

Hospital Clinic CHC CHP MCHP

Level-6 organisation units (community health workers, HTR/ETR workers operating under an MCHP) are fetched in parallel. A childToL5 map is built to link each Level-6 UID to its Level-5 MCHP parent, so community data reported at Level-6 is correctly attributed to the parent facility.

2 · Data Elements

Thirteen data elements are queried via /api/dataValueSets across two DHIS2 datasets:

HF01 — Outpatient Morbidity  (Dataset UID: ij1VNJwxKCh)

Data entered directly at Level-5 by facility staff. Queried with children=true so Level-6 CHW/ETR community records are automatically included.

3 · Reporting Status Rule

Reported = 1  —  A facility is marked as Reported for a given month if the sum of all values across any of the 13 data elements above is greater than zero for that facility-month combination.

Not Reported = 0  —  If no non-zero value is found across all 13 data elements for that facility and month, the facility is marked as Not Reported.

HF04 community data entered at Level-6 is rolled up to the parent Level-5 MCHP before this check. A single non-zero community record is sufficient to mark the MCHP as reported.

4 · Aggregation

Facility-month records are aggregated to district-month, district-year, chiefdom-month, and chiefdom-year levels. The reporting rate at each level is: Reported facility-months ÷ Total facility-months × 100. Geographic rates are computed from the chiefdom-month aggregates and mapped to GeoJSON boundaries.

5 · Technical Details

DHIS2 instance: sl.dhis2.org/hmis23  ·  API endpoint: /api/analytics.json with dimension=ou:LEVEL-5 (DHIS2 pre-aggregates all data server-side, Level-6 CHW/ETR community records included automatically)  ·  Org unit level: 5 (facility)  ·  Period type: Monthly  ·  Developed by: Informatics Consultancy Firm – Sierra Leone (ICF-SL) for the National Malaria Control Programme (NMCP).
