Malaria Analytics Dashboard

Comprehensive Malaria Surveillance — Sierra Leone HMIS

How It Works
1
Fetch Facility Registry
Loads all Level-5 health facilities from DHIS2 with district & chiefdom hierarchy. Classifies each as Hospital, Clinic, CHC, CHP or MCHP — all other types excluded.
2
Build Query Plan
Packages 48 malaria indicator de.coc pairs and the selected periods into optimised batches for the DHIS2 analytics API — typically just 2–4 API calls total.
3
Fetch & Clean Analytics
Pulls pre-aggregated data from DHIS2. Applies Sierra Leone reporting rules: microscopy excluded for CHP & MCHP; admissions excluded for CHC, CHP & MCHP.
4
Derive Indicators
Computes total suspected, total tested (RDT + microscopy), confirmed cases, ACT treatments, admissions, deaths and cascade rates per facility per month.
5
Render Dashboard
Builds interactive charts, summary cards and district/facility filters. All charts show count with percentage on hover.
Select Period
Choose the date range then click Run Pipeline to fetch data from DHIS2.
